# 🧠 Exploring Biomedical Summarization Pipelines with BART, BioBART, and FalconsAI 📚

This project began as an NLP research experiment focused on biomedical text summarization using the `ccdv/pubmed-summarization` dataset. The initial objective was straightforward: compare the performance of general-purpose and domain-specialized transformer models on long-form scientific articles. However, as the implementation evolved, the project gradually shifted into a broader exploration of how model alignment, preprocessing strategies, and pipeline architecture interact in practical summarization systems. Rather than treating summarization as a purely model-centric task, the experimentation process revealed how strongly performance depends on the surrounding engineering decisions.

The pipeline was structured incrementally, beginning with dataset loading and cleaning, followed by token-based chunking designed to handle long biomedical documents exceeding transformer context limits. Chunking was implemented using `AutoTokenizer.from_pretrained("facebook/bart-base")` with `max_tokens=1024`, `overlap=50`, and `min_tokens=50` to preserve contextual continuity between segments. The first stage of experimentation focused on BioBART as a domain-oriented variant intended to test whether biomedical specialization alone could improve summarization quality. During this phase, several artifact-generation issues emerged, including formatting inconsistencies and noisy output patterns, which motivated the creation of a dedicated cleaning function to normalize punctuation, remove citation remnants, and reduce repeated symbol artifacts before final aggregation.

The comparative evaluation was organized around three distinct model categories: BART as a generalist baseline, BioBART as a domain-specialized variant, and FalconsAI Medical Summarization as a task-aligned biomedical summarizer. This structure allowed the analysis to move beyond simple score comparison into a more meaningful evaluation of domain adaptation versus task alignment. ROUGE metrics were computed across all three models, complemented by qualitative inspection focused on coherence, redundancy, factual consistency, and terminology handling. The observed pattern aligned closely with the initial hypothesis: BioBART underperformed despite domain specialization, BART remained stable and coherent across chunked summaries, and FalconsAI achieved the strongest overall performance in biomedical compression and terminology usage, although its outputs occasionally exposed limitations introduced by chunk-level summarization and reconstruction.

One of the most important insights of the project emerged from these limitations. The experiments demonstrated that domain knowledge alone does not guarantee summarization quality when the model is not explicitly aligned with the downstream task. At the same time, even strong task-aligned models remained constrained by the architecture of the preprocessing pipeline itself, particularly the fragmentation introduced by token chunking. This shifted the interpretation of the project away from identifying a “best model” and toward understanding summarization as a systems problem involving tokenizer strategy, chunk boundaries, reconstruction logic, and generation stability. The evaluation therefore became less about achieving a terminal result and more about identifying where the next iteration of improvement should occur.

The final stage of the project transformed the research workflow into an interactive application using Gradio, enabling users to explore biomedical summarization dynamically through a unified interface. In addition to the implementation itself, the project incorporated comparative visualizations of ROUGE performance and a technical stack built around Hugging Face Transformers, PyTorch, Evaluate, Matplotlib, Pandas, and Gradio. References included the PubMed summarization dataset and prior work on abstractive summarization architectures, particularly the transformer-based approaches introduced in contemporary NLP literature. Future improvements would focus primarily on semantic chunking strategies, hierarchical summarization architectures, and evaluation methods extending beyond ROUGE in order to better capture factual consistency and long-document coherence in biomedical contexts.
